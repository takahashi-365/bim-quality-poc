# Local LLM プロンプトテンプレート

## 目的

このドキュメントは、第3段階A：Local LLM Explanation Demo で使用するプロンプトテンプレートを定義する。

本プロンプトの目的は、第2段階PoCで生成した構造化済みのBIM品質コンテキストを、BIM担当者が確認しやすい説明文へ変換できるかを検証することである。

本プロンプトは、LLMに設計判断、施工判断、Revitモデル修正の最終判断を行わせるためのものではない。

LLMの出力は参考情報として扱い、最終判断はBIM担当者または人間のレビュー担当者が行う。

---

## 入力範囲

本プロンプトでは、以下の情報を入力として使用する。

* ElementId 1件分の AI Context
* 該当 RuleId の Fix Guide
* Severity
* QualityScore
* AI Readiness Score
* HumanReviewRequired
* BlockingRuleIds

`ai_context_v002.md` 全文をLLMへ渡すのではなく、ElementId 1件単位に切り出して使用する。

---

## プロンプトテンプレート

```text
あなたはBIMデータ品質確認を補助するアシスタントです。

以下の AI Context と Fix Guide をもとに、BIM担当者向けの説明文を作成してください。

重要な前提：
- 設計判断、施工判断、モデル修正の最終判断は行わないでください。
- 入力情報に基づいて説明してください。
- 入力に含まれない情報は「入力情報からは判断できません」と表現してください。
- 不明な内容を推測しすぎないでください。
- HumanReviewRequired が True の場合は、人間確認が必要であることを明記してください。
- 修正方針は「案」として表現してください。
- 回答の根拠として参照した RuleId、Severity、AI Readiness Score、FixGuide を明記してください。
- 出力の最後に、LLM回答は参考情報であり、最終判断はBIM担当者が行うことを明記してください。
- 出力はBIM担当者が確認しやすい日本語にしてください。

出力形式：
1. 対象要素の概要
2. 検出された問題
3. AI活用上の影響
4. 修正優先度と理由
5. 人間が確認すべき項目
6. 修正方針案
7. 根拠として参照した情報
8. 注意点

AI Context：
{AI_CONTEXT}

Fix Guide：
{FIX_GUIDE}
```

---

## 入力変数

| 変数             | 内容                        |
| -------------- | ------------------------- |
| `{AI_CONTEXT}` | ElementId 1件分の AI Context |
| `{FIX_GUIDE}`  | 該当 RuleId の Fix Guide     |

---

## 出力要件

LLMの出力には、以下の項目を含める。

1. 対象要素の概要
2. 検出された問題
3. AI活用上の影響
4. 修正優先度と理由
5. 人間が確認すべき項目
6. 修正方針案
7. 根拠として参照した情報
8. 注意点

---

## 制約条件

LLMは以下の制約に従う。

* 入力された AI Context と Fix Guide の範囲内で説明する。
* 入力にないプロジェクト条件を推測しない。
* 設計内容や施工方法の妥当性を判断しない。
* Revitモデルの自動修正を指示しない。
* 修正方針は最終指示ではなく「案」として表現する。
* HumanReviewRequired が True の場合は、人間確認が必要であることを明記する。
* LLM出力は参考情報であることを明記する。
* 最終判断はBIM担当者が行うことを明記する。

---

## 人間レビュー観点

生成された説明文は、以下の観点で人間が確認する。

| 評価項目                         | 結果                | コメント |
| ---------------------------- | ----------------- | ---- |
| RuleIdに沿って説明できているか           | OK / Partial / NG |      |
| Severityを誤解していないか            | OK / Partial / NG |      |
| AI Readiness Scoreを正しく扱えているか | OK / Partial / NG |      |
| HumanReviewRequiredを明記できているか | OK / Partial / NG |      |
| FixGuideを自然文に変換できているか        | OK / Partial / NG |      |
| 根拠として参照した情報を明示できているか         | OK / Partial / NG |      |
| 余計な設計判断をしていないか               | OK / Partial / NG |      |
| 存在しない情報を推測していないか             | OK / Partial / NG |      |
| BIM担当者向けに分かりやすいか             | OK / Partial / NG |      |
| 最終判断はBIM担当者が行うと明記できているか      | OK / Partial / NG |      |

---

## 失敗時の扱い

生成結果に誤解、過剰推測、設計判断に近い表現、入力にない情報の補完が含まれた場合も、MVPとしては失敗とは扱わない。

その場合は、以下の改善点として記録する。

* AI Context v002 の情報不足
* Fix Guide の粒度不足
* プロンプトの制約不足
* 入力範囲の切り出し方法
* LLM出力の評価観点
* HumanReviewRequired の表現方法
* 「入力情報からは判断できません」と言わせる制約の不足
* 最終判断はBIM担当者が行うことの明示不足

---

## 備考

このプロンプトは、Local LLM実験用である。

目的は、LLMの性能評価ではない。
目的は、`AI Context v002` が説明生成用の構造化コンテキストとして利用できるかを確認することである。

生成結果は Raw LLM Output として保存する。
人間による評価は、Raw LLM Output とは分けて記録する。
