# 第3段階A：Local LLMでAI Context説明生成デモ 計画

## 目的

第2段階PoCで生成した `AI Context v002` と `Fix Guide Markdown` を、ローカルLLMに読み込ませ、BIM担当者向けの説明文を生成できるかを検証する。

本検証の目的は、LLMに設計判断や施工判断をさせることではない。
RuleId、Severity、QualityScore、AI Readiness Score、HumanReviewRequired、FixGuideなど、既存PoCで構造化した情報をもとに、BIM担当者が確認しやすい説明文へ変換できるかを確認する。

重要なのは、LLMの性能比較ではなく、`AI Context v002` がLLMに渡す構造化コンテキストとして利用可能かを検証することである。

---

## 位置づけ

本作業は、新規PoCではなく、既存の `BIM Data Quality & AI Readiness Assessment PoC` の第3段階拡張として実施する。

第2段階までの流れ：

```text

↓
CSV変換
↓
データクレンジング
↓
RuleIdベース品質チェック
↓
QualityScore
↓
AI Readiness Score
↓
AI Context v002
↓
Fix Guide Markdown
↓
Streamlit表示
```

第3段階Aでは、以下を追加する。

```text
AI Context v002
+ Fix Guide Markdown
↓
Local LLM
↓
BIM担当者向け説明文生成
↓
人間確認前提の修正方針案
```

## 第3段階Aでやること

* ローカルLLM実行環境を準備する
* `ai_context_v002.md` の一部を入力として使う
* `fix_guides_v001.md` の一部を入力として使う
* 入力は原則としてElementId 1件単位に限定する
* 固定プロンプトを作成する
* BIM担当者向けの説明文を生成する
* 生成結果をMarkdownに保存する
* 生成結果をRaw LLM Outputとして記録する
* 生成結果に対するHuman Reviewを別に記録する
* 生成結果を評価表で確認する
* 生成結果の良かった点・限界・注意点を記録する
* 使用したローカルLLM環境、モデル名、量子化形式、実行環境を記録する
* READMEにはMVP完了後に小さく追記する
* StreamlitやPDFへの反映は初回MVPでは必須としない

## 入力範囲の方針

初回MVPでは、`ai_context_v002.md` 全文をローカルLLMに渡さない。

入力は、以下に限定する。

```text
ElementId 1件分のAI Context
+ 該当RuleIdのFix Guide
```

理由：

* 入力が短くなり、誤読を減らせる
* 出力評価がしやすい
* ElementId単位の説明生成に向いている
* 将来的なRAGにおけるElement chunk設計にもつながる
* LLMが存在しない情報を推測するリスクを下げられる

## プロンプト方針

プロンプトでは、以下を明示する。

* あなたはBIMデータ品質確認を補助する説明文を作成する
* 設計判断、施工判断、モデル修正の最終判断はしない
* 入力されたRuleId、Severity、AI Readiness Score、FixGuideに基づいて説明する
* 入力に含まれない情報は「入力情報からは判断できません」と表現する
* 不明な内容は推測しすぎない
* HumanReviewRequiredがTrueの場合、人間確認が必要であることを明記する
* 修正方針は「案」として表現する
* 回答の根拠として参照したRuleId、Severity、AI Readiness Score、FixGuideを明記する
* 出力の最後に、LLM回答は参考情報であり、最終判断はBIM担当者が行うことを明記する
* 出力はBIM担当者が確認しやすい日本語にする

## 失敗時の扱い

生成結果に誤解、過剰推測、設計判断に近い表現、入力にない情報の補完が含まれた場合も、MVPとしては失敗とは扱わない。

その場合は、以下の改善点として記録する。

* AI Context v002の情報不足
* Fix Guideの粒度不足
* プロンプトの制約不足
* 入力範囲の切り出し方法
* LLM出力の評価観点
* HumanReviewRequiredの表現方法
* 「入力情報からは判断できません」と言わせる制約の不足
* 最終判断はBIM担当者が行うことの明示不足

本検証では、LLM回答の完成度よりも、AI Context v002をLLMに渡す構造化コンテキストとして利用できるか、またどのような制約があるかを確認することを重視する。
