# BIM品質チェック・ルール改善支援PoC

# 概要

このPoCは、Revit/BIMデータの品質チェック結果をRuleIdで管理し、Pythonによる自動チェック、Power BIによる可視化、生成AI活用前提のプロンプト生成までを一連の流れとして検証したものです。

目的は、BIM導入支援で課題になりやすい「パラメータ未入力」「分類コード未整備」「命名規則違反」を検出し、BIMデータ品質を可視化・改善支援できる仕組みを検討することです。

## このPoCで示したいこと

- BIM標準化ルールをRuleIdとしてデータ化できる
- Revit/BIMデータをPythonで品質チェックできる
- チェック結果をPower BIで可視化できる
- RuleIdを起点に、改善方針や業務影響を参照できる
- 生成AIへ渡す情報を制限し、根拠付き回答を行う設計にできる
- APIキーや従量課金を使わず、生成AI連携前段階の設計を検証できる

## 目的

BIM標準化ルールをRuleIdでデータ化し、Revit/BIMデータをPythonで品質チェックし、Power BIで可視化し、将来的に生成AIで改善方法を確認できるPoCを作成する。

## 使用データ

Autodesk日本仕様 意匠サンプルモデル Revit 2024を検証用データとして使用する。
実案件データ・社外秘データは使用しない。

## RuleId連携型 プロンプト生成デモ 初期版

作成ファイル：
06_ai_demo/ruleid_prompt_generator_demo.py

参照データ：
02_rule_master/bim_rule_master_v001.csv

出力先：
06_ai_demo/generated_prompts

目的：
RuleIdをキーにルールマスタから該当ルールを取得し、生成AIに渡すためのプロンプトを自動生成する。

生成済みプロンプト：
06_ai_demo/generated_prompts/prompt_R-001_20260517_172551.txt
06_ai_demo/generated_prompts/prompt_R-002_20260517_172615.txt
06_ai_demo/generated_prompts/prompt_R-003_20260517_172644.txt

確認結果：
R-001、R-002、R-003 を入力し、それぞれ該当ルールの内容をもとに生成AI用プロンプトを作成できることを確認した。
作成したプロンプトはtxtファイルとして保存済み。

プロンプトに含めた条件：
・参照ルールに書かれている内容を優先する
・参照元RuleIdを必ず明記する
・資料にない内容は断定しない
・自動修正を前提にしない
・設計判断、施工判断、モデル修正の最終判断は人間が行う

補足：
初期版ではOpenAI APIなどを使って生成AIを直接呼び出さず、AIに渡すプロンプトを生成する構成とする。
これにより、APIキー管理や従量課金を避けながら、RuleIdを起点に参照元を限定した生成AI活用設計を示す。
